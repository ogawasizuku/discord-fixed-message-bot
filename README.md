# Discord定型文Bot

Discordのスラッシュコマンドで、サーバーごとに定型文・Poll送信を管理するBotです。

## 主な機能

- サーバーごとに定型文を登録・管理
- 指定時刻にPoll形式で自動送信
- 定型文の一時停止・再開
- 定型文の質問・時刻・制限時間・選択肢を後から変更
- 送信先チャンネルの変更
- 送信履歴の確認
- systemdによるVPS常時稼働対応
- `.env` によるDiscord Bot Token管理

## コマンド一覧

| コマンド | 用途 |
|---|---|
| `/定型文追加` | 新しい定型文・Pollを登録します。質問、送信時刻、制限時間、選択肢、複数回答、@everyoneメンション有無を設定できます。 |
| `/定型文削除` | 登録済みの定型文を削除します。 |
| `/定型文一覧` | このサーバーに登録されている定型文を一覧表示します。 |
| `/定型文即送信` | 登録済みの定型文を、予約時刻を待たずにすぐ送信します。 |
| `/定型文テスト送信` | 登録済みの定型文をテスト送信します。運用前の確認に使います。 |
| `/定型文一時停止` | 指定した定型文の自動送信を一時停止します。 |
| `/定型文再開` | 一時停止中の定型文を再開します。 |
| `/定型文全停止` | このサーバーに登録されている全ての定型文を一時停止します。 |
| `/定型文全再開` | このサーバーに登録されている全ての定型文を再開します。 |
| `/定型文時刻変更` | 登録済み定型文の送信時刻を変更します。 |
| `/定型文制限時間変更` | Pollの回答制限時間を変更します。 |
| `/定型文質問変更` | 登録済み定型文の質問文を変更します。 |
| `/定型文選択肢変更` | Pollの選択肢を変更します。 |
| `/定型文複数回答変更` | 複数回答を許可するかどうかを変更します。 |
| `/定型文メンション変更` | @everyoneメンションを付けるかどうかを変更します。 |
| `/定型文チャンネル変更` | 定型文の送信先チャンネルを変更します。 |
| `/定型文コピー` | 既存の定型文をコピーして、新しい定型文を作成します。 |
| `/定型文送信履歴` | 定型文の最終送信日や次回送信時刻を確認します。 |

## 使い方の例

### 1. 定型文を追加する

Discord上で `/定型文追加` を実行し、以下を入力します。

- 名前
- 質問文
- 送信時刻
- 制限時間
- 選択肢
- 複数回答の有無
- @everyoneメンションの有無

### 2. 登録内容を確認する

```text
/定型文一覧
```

### 3. 送信前に確認する

```text
/定型文テスト送信
```

### 4. 一時停止する

```text
/定型文一時停止
```

### 5. 再開する

```text
/定型文再開
```

## ローカル起動

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
# .env に DISCORD_TOKEN を設定
python main.py
```

Windows PowerShellの場合:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
copy .env.example .env
python main.py
```

## 常時稼働の推奨構成

GitHubはコード置き場として使い、Botの常時稼働はVPSで行います。

無料・低コスト運用なら Oracle Cloud Always Free などのLinux VPS + systemd が安定です。

詳しい導入手順は以下を参照してください。

- [セットアップ手順](docs/SETUP.md)
- [運用メモ](docs/OPERATIONS.md)

## 注意

- `.env` はGitHubへ上げないでください。
- `bot_config.json` は実データなので、原則GitHubへ上げないでください。
- サーバー移行時だけ、必要に応じて `bot_config.json` を安全な方法でコピーしてください。
- `bot_config.example.json` は公開用のサンプル設定です。
- 送信時刻はサーバーのタイムゾーンに依存します。日本時間で運用する場合はVPS側を `Asia/Tokyo` に設定してください。
